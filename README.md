# 🎬 ارشیو فیلم‌های من

وب‌اپ شخصی برای آرشیو فیزیکی فیلم‌ها. فیلم‌هات رو از روی **فایل اکسل** وارد می‌کنی و
با کلیک روی هر فیلم، همه جزئیاتش (عکس، کارگردان، بازیگران اصلی، محل فیزیکی روی
قفسه و…) باز می‌شه.

ساخته‌شده با **React** (جلو) و **Node.js / Express** (پشت).

---

## ✨ ویژگی‌ها

- ایمپورت آسان از اکسل (ستون‌های فارسی و انگلیسی پشتیبانی می‌شن)
- جستجو بر اساس نام فیلم، کارگردان یا بازیگر
- فیلتر بر اساس ژانر و قفسه + مرتب‌سازی
- نمایش محل فیزیکی فیلم: **قفسه / ردیف**
- کلیک روی فیلم → نمایش کامل با عکس، بازیگران و کارگردان
- (اختیاری) غنی‌سازی خودکار از **TMDB** برای عکس/بازیگر/کارگردان

---

## 📋 ساختار فایل اکسل

ستون‌های اجباری فقط **نام فیلم**، **قفسه** و **ردیف** هستن؛ بقیه اختیاری‌ان.
برای دیدن ستون‌ها یا دانلود قالب آماده:

```
GET /api/template        →  دانلود فایل film-archive-template.xlsx
```

| ستون (فارسی) | ستون (انگلیسی) | توضیح |
|---|---|---|
| نام فیلم | title | **ضروری** |
| قفسه | shelf | ضروری — مثلاً A |
| ردیف | row | ضروری — مثلاً 3 |
| نام اصلی | originalTitle | نام لاتین |
| کارگردان | director | |
| بازیگران | cast | با ویرگول جدا کنید |
| سال | year | |
| ژانر | genre | با ویرگول جدا کنید |
| امتیاز | rating | ۰ تا ۱۰ |
| زمان | runtime | به دقیقه |
| کشور | country | |
| خلاصه | synopsis | |
| لینک پوستر | poster | آدرس اینترنتی عکس |
| وضعیت تماشا | watched / Watch Status | اختیاری — `بله` / `Yes` / `1` برای تماشا‌شده و `خیر` / `No` / `0` برای دیده‌نشده |

> نکته: ایمپورت جدید با آرشیو قبلی **ادغام** می‌شه (بر اساس نام فیلم). فیلم تکراری
> آپدیت، و فیلم جدید اضافه می‌شه. اگر ستون وضعیت تماشا خالی باشد، وضعیت قبلی آن
> فیلم تغییر نمی‌کند.

## 🚀 اجرا در حالت توسعه (روی سیستم خودت)

```bash
npm install
npm run dev
# سرور روی http://localhost:3001 و رابط کاربری روی http://localhost:5173
```

## 🌐 ساخت نسخه نهایی و اجرا (برای هاست)

```bash
npm install
npm run build      # خروجی در پوشه dist
npm start          # سرور هم API رو می‌ده هم سایت رو سرو می‌کنه
# http://localhost:3001
```

قالب اکسل رو هم می‌تونی جداگانه بسازی:
```bash
npm run template   # خروجی: film-archive-template.xlsx
```

## 🔑 غنی‌سازی خودکار با OMDb (اختیاری)

اگر کلید رایگان [OMDb](https://www.omdbapi.com/) رو داشته باشی، هر فیلمی که
از **Add Film** اضافه یا از اکسل ایمپورت می‌کنی، به‌طور خودکار از روی نام فیلم
جستجو می‌شه. فقط فیلدهای ناقص (پوستر، سال، کارگردان، بازیگران، ژانر، امتیاز،
مدت، کشور، خلاصه، رده‌بندی و شناسهٔ IMDb) پر می‌شن و اطلاعاتی که خودت وارد کردی
هرگز جایگزین نمی‌شن:

```bash
OMDB_API_KEY=کلید_تو npm start
```

برای فیلم‌هایی که قبل از فعال‌کردن این قابلیت اضافه شده‌اند، در پنجرهٔ Edit روی
**Auto-fill missing details** بزن. برای تکمیل یک فهرست بزرگ (مثل فهرست PDF)، از
دکمهٔ **Fill missing details** در بالای آرشیو استفاده کن؛ این کار درخواست‌ها را
در دسته‌های کوچک انجام می‌دهد تا اطلاعات ثبت‌شده جایگزین نشوند. اگر کلید ندی،
فیلم همچنان ذخیره می‌شه اما هیچ تماسی با اینترنت برقرار نمی‌شه.

---

## ☁️ دیپلوی روی هاست (مثلاً Render / Railway)

1. ریپو رو به GitHub پوش کن.
2. سرویس جدید بساز (Build: `npm install && npm run build`، Start: `npm start`).
3. متغیر محیطی `PORT` رو هاست معمولاً خودش تنظیم می‌کنه؛ `OMDB_API_KEY` رو هم
   اختیاری اضافه کن.
4. فایل راکت (اگه خواستی) رو در همین پوشه بذار (`start` از همون استفاده می‌کنه).

```

├── index.html
├── vite.config.js
├── server/
│   ├── index.js        # سرور اکسپرس + API

│   ├── make-template.js# ساخت فایل قالب اکسل
│   └── data/
│       └── films.json  # داده‌های آرشیو (بعد از ایمپورت اینجا ذخیره می‌شه)
└── src/
    ├── main.jsx
    ├── App.jsx
    ├── styles.css
    └── components/
        ├── Header.jsx
        ├── FilmGrid.jsx
        ├── FilmCard.jsx
        └── FilmModal.jsx
```

داده‌ها در `server/data/films.json` ذخیره می‌شن — برای بکاپ فقط کافیه این فایل
رو نگه داری.
